## Fix: Event loop conflict + Cache-hit crash

### Root Cause
1. **Event loop conflict**: `engine` و `AsyncSessionLocal` سطح ماژول ساخته شدن (`db/session.py` lines 4-5). وقتی `run_async()` یه event loop جدید تو thread جدید می‌سازه، engine قبلی هنوز به loop دیگه‌ای bind شده → `Future attached to a different loop`
2. **Cache-hit crash**: وقتی فایل قبلاً تحلیل شده، `df` (DataFrame تراکنش‌ها) هیچوقت ساخته نمیشه ولی display section ازش `df.copy()` می‌زنه → `NameError`

### Fix 1: `db/session.py` — Lazy engine creation
Engine و session factory رو lazy بسازیم (per-call)، نه module-level. هر بار که `get_session()` صدا زده بشه، engine و session factory جدید ساخته بشه.

```python
# db/session.py
import config
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker

def _make_engine():
    return create_async_engine(config.DATABASE_URL, echo=False)

def get_session():
    engine = _make_engine()
    Session = async_sessionmaker(engine, expire_on_commit=False)
    return Session()

async def init_db():
    engine = _make_engine()
    from db.models import Base
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    await engine.dispose()
```

### Fix 2: `app.py` — Update DB calls + Fix cache-hit path
- `_check_db_cache()` و `_save_to_db()` از `get_session()` استفاده کنن (بجای `AsyncSessionLocal`)
- Cache-hit path: تراکنش‌ها رو هم از DB بخونه و `df` بسازه (یا بخش transactions جدول رو skip کنه)
- ساده‌ترین: وقتی cache-hit میشه، display section جدول تراکنش‌ها رو skip کنه (چون `df` نداریم)

### Files changed:
1. `db/session.py` — lazy engine, `get_session()` function
2. `db/crud.py` — add `get_transactions_by_report_id()` for cache-hit path  
3. `app.py` — update imports, use `get_session()`, reconstruct `df` on cache-hit
